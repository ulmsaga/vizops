## 스킬 목록

### safe-security-fix
보안 취약점을 탐지하고 실제로 수정합니다.

**실행 절차:**
1. `git checkout main && git pull` — 최신 코드 확인
2. `git checkout -b fix/security-$(date +%Y%m%d)` — 작업 브랜치 생성
3. 전체 코드에서 보안 취약점 스캔:
   - 하드코딩된 API 키, 비밀번호, 토큰 (`.env` 제외 — 이미 gitignore)
   - SQL Injection 취약점 (raw query + user input)
   - XSS 취약점 (unsanitized user input in HTML)
   - CSRF 미적용 폼
   - `eval()`, `exec()` 같은 위험한 함수 사용
4. 발견된 취약점 실제 수정 (파일 Edit)
5. `git add -A && git commit -m "fix(security): apply safe-security-fix"`
6. `gh pr create --title "fix(security): safe-security-fix by Jarvis" --body "## Summary\n보안 취약점 자동 수정\n\n🤖 Generated by Jarvis"`
7. PR URL 출력

**완료 보고 형식:**
```
✅ safe-security-fix 완료
- 수정 파일: N개
- 건너뜀: M건 (이미 안전)
- PR: https://github.com/ulmsaga/vizops/pull/XXX
```

### generate-docs
코드 문서를 자동 생성합니다.

**실행 절차:**
1. `git checkout main && git pull`
2. `git checkout -b docs/auto-$(date +%Y%m%d)`
3. 모든 함수/클래스에 Google 스타일 docstring 추가
4. README.md 업데이트 (새 기능, API 변경사항 반영)
5. `git add -A && git commit -m "docs: auto-generate docstrings and README"`
6. `gh pr create --title "docs: auto-generated documentation by Jarvis" --body "🤖 Generated by Jarvis"`
7. PR URL 출력

### code-review
코드를 리뷰하고 개선점을 제안합니다.

**실행 절차:**
1. 전체 코드 분석 (또는 지정된 PR diff)
2. 다음 항목 검토:
   - 코드 품질 및 가독성
   - 버그 가능성
   - 보안 이슈
   - 성능 문제
3. 구체적인 개선 제안 (파일명:줄번호 형식으로)
4. GitHub PR comment로 결과 게시 (PR URL이 있는 경우)

## 작업 규칙

- **반드시 실제로 파일을 수정할 것** — 설명만 하지 말 것
- **PR 없으면 완료 아님** — git commit + gh pr create까지 완료
- **PR 링크 반드시 출력** — 마지막에 PR URL 포함
- **오류 발생 시 정확한 에러 메시지 출력**
- **작업 브랜치 사용** — main에 직접 push하지 않음
